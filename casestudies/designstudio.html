01

Context & the Imperative

UI Design Studio is a centralised portal for managing and distributing reusable UI assets — templates, design stencils, icons, components, and themes — built for HCL Technologies' internal delivery teams.

The core insight was simple: every project was rebuilding the same UI from scratch. By creating a shared, searchable library with project management, contribution workflows, and advanced search, teams could start with reviewed, cross-browser compatible components instead of blank canvases.

The platform covers three distinct experiences — a public-facing portal for discovery and downloads, an admin console for full library management, and a contributor console for community-sourced additions — all unified under a single design system.

02

The Problem

Delivery

Every project starting from zero

  • Teams rebuilt identical UI components on every engagement
  • No shared baseline — inconsistent visual standards across accounts
  • Duplicated effort across parallel delivery teams
  • No mechanism to share or reuse approved work
  • Weeks lost on UI scaffolding before real work began

Quality

No standard, no review, no consistency

  • Different browser compatibility levels across projects
  • No cross-browser testing before client delivery
  • Responsive design handled inconsistently or not at all
  • Design stencils and icons scattered across local drives
  • No approval or review gate for new UI patterns

Visibility

No way to discover what already existed

  • No searchable repository of existing components
  • Teams unaware of work done by other teams
  • No download tracking or usage insight
  • No community contribution pathway
  • Knowledge siloed at the individual, not the organisation

"We kept building the same things over and over. There was no way to know if another team had already solved the same problem — and no place to put it even if they had."

HCL delivery team feedback — discovery phase
03

Features

The Studio was designed to solve the reuse problem end-to-end — from discovery and download through contribution, review, and project-level management.

Discovery

Advanced asset search

Search templates and components by business domain, browser support, platform, form factor, responsive flag, and category — not just by name.

Library

Ready-to-use component repo

Pre-reviewed, cross-browser compatible UI components built on Bootstrap, jQuery, Backbone, and Knockout — ready to download and customise immediately.

Projects

Project-scoped asset management

Teams manage project-specific templates, themes, stencils, and components — assets organised per delivery rather than shared in one undifferentiated pool.

Contribution

Community contribution workflow

Any user can register as a contributor and submit templates, icons, and themes. Admin approval gates ensure quality before assets enter the shared library.

Responsive

RWD-ready components

All templates and components built to Responsive Web Design standards — tested across Android, iOS, tablet and phone before publishing to the library.

Tracking

Download & usage analytics

Download tracking, usage analytics, feedback loops, and support tickets give admins visibility into which assets are being used — and what's missing.

04

System Design

Rather than building three separate applications, the Studio was architected as a single platform with role-based views — reducing build and maintenance overhead while keeping each portal's UX tightly scoped to its user type.

Public Portal

Discovery & Downloads

The primary library for all internal delivery teams. Browse, search, preview, and download production-ready UI assets — filtered by platform, domain, framework, and responsive support.

Admin Console

Full Library Management

Approve contributions, manage asset metadata, set visibility rules, view analytics, and handle support tickets — the single control plane for library quality and governance.

Contributor Console

Community Submissions

Any verified HCL employee can register as a contributor and submit new components, themes, or icons. Assets enter a review queue before appearing in the public library.

Shared Infrastructure

One repository, three experiences

All three portals share the same asset repository, search engine, and notification system. A contribution accepted by admin instantly appears for the next engineer searching by component type.

05

Process

A user-centred design approach grounded in delivery team needs — validated through iterative releases rather than a single big-bang launch.

01 · Discovery

Understand the reuse gap

  • Stakeholder interviews across delivery teams
  • Audit of existing component sprawl
  • Identification of high-frequency UI patterns
  • Framework and tech stack analysis

02 · Architecture

Design the system

  • Three-portal role model definition
  • Information architecture per user type
  • Contribution and approval workflow design
  • Search taxonomy and metadata schema

03 · Design & Build

Prototype and validate

  • Lo-fi wireframes per portal experience
  • Usability testing with delivery team users
  • Visual design system definition
  • PHP build with Bootstrap/jQuery foundation

04 · Release

Alpha → Beta → Final

  • Alpha: core library with pilot teams
  • Beta: contribution and admin workflows
  • Final: full platform rollout across HCL
  • Post-launch: analytics and iteration

07

Expected Outcomes

60%

Effort reduction in UI delivery

Teams starting from the Studio library rather than a blank canvas reduced front-end development effort by up to 60%. With pre-reviewed, cross-browser components as the baseline, the quality floor was raised across every project simultaneously — not just the ones lucky enough to have experienced engineers.

Further outcomes across the platform

10-day delivery baseline — the combination of pre-reviewed components, advanced search, and project scoping collapsed typical multi-week UI build phases into a reliable ten-day starting point.

Consistency across all accounts — a single shared library meant cross-browser standards, accessibility baselines, and responsive behaviour were inherited by default, not enforced project by project.

Community-driven growth — contribution workflows ensured the library grew through the collective expertise of HCL's delivery teams, not just a central design team.

Full usage visibility — download tracking and analytics gave library maintainers the data to prioritise new components based on actual demand.

08 · Reflections

What worked

  • Role-based architecture from day one. Designing three distinct portals on a shared foundation meant each user group got a focused experience without tripling the build cost.
  • Phased release strategy. Alpha → Beta → Final allowed real delivery team feedback to shape contribution and admin workflows before full rollout.
  • Advanced search as the core differentiator. Filtering by browser support and responsive flag — not just category — made the library genuinely useful rather than just comprehensive.
  • Community contribution model. Giving contributors a structured pathway meant the library grew organically rather than requiring a dedicated content team to scale it.

What I'd do differently

  • Earlier usability testing with real delivery teams. Heuristic review shaped the initial design, but task-based sessions mid-build would have surfaced workflow gaps before Beta.
  • Richer analytics from Alpha. Download tracking gave useful signal, but session-level analytics from the first release would have accelerated decisions about which asset types to prioritise.
  • Stronger onboarding for new contributors. The submission workflow assumed familiarity with metadata requirements — a guided submission flow would have reduced admin review overhead significantly.
  • Mobile-first the admin console. Admin and contributor portals were built desktop-first — a decision that limited flexibility for team leads needing to review or approve on the go.
The problem wasn't that teams lacked skill — it was that there was no system for them to build on. The Studio didn't just save time; it raised the floor for every project that came after it.
